# 데이터 분석 프로그램 별 장단점 

## 1. R 
장점 
- 컴퓨터 과학 수준에서 프로그래밍하기가 무척 쉽다는 것 
- 개선된 속도로 인한 빠른 데이터 처리 
- 다양한 데이터 집합, 도구, 소프트웨어 패키지를 포괄하는 일종의 접착 언어 역할 
- 확장성이 뛰어나고 개발자에게 데이터 분석을 위한 자체 도구와 방법을 만들 수 있는 풍부한 기능 제공 
- 기계 학습 분야에서 학계와의 강력한 연계성 
- 접근성이 쉬운 언어 
- 방대한 양의 패키지 
- 즉시 사용 가능한 테스트 SET 제공  
- windows, Linux, Unix, Mac OS 등 대부분의 개발환경 지원 
- 하둡(Hadoop) 분산처리환경 지원

단점
- 프로그래머용이 아닌 데이터 지향적인 문제를 해결하려는 사람에게만 적합한 언어 

## 2. Python
장점
- IPython Notebook을 통한 수월한 데이터 작업 
- 특정 주제나 용도에 국한되지 않은 언어 
- 비교적 쉽고 직관적
- 플랫폼 독립적
- 동적 타이핑(dynamic typed) 대화형 언어 : 변수 타입을 따로 지정하지 않고 실행할 때 변수의 타입을 자동으로 검사한다.(Duck typing)
- 높은 접근성 
- 오픈소스 언어이며, 다른 언어로 제작된 모듈들도 포함시켜 사용이 가능하다.
- 사용의 단순함과 편리함을 위해 접근제어 없이 객체, 구조체 멤버에 무제한적 접근이 가능 
- 절자척, 객체지향, 함수형, 관점형 프로그래밍이 모두 가능하다.
- 잘 설계된 내장 라이브러리와 표준 라이브러리, 서드파티 오픈소스 라이브러리와 방대한 모듈 

단점
- R에 비해 적은 패키지 수 
- 상대적으로 아쉬운 시각화 기능
- 인터프리터 언어로 인해 종종 컴파일 언어보다 느린 경우가 생김
- 모바일 컴퓨팅과 브라우저 부재 

## 3. SCALA
장점
- 짧은 코드, 높은 생산성
- 높은 확장성 : 함수형 언어와 객체 지향 언어의 개념을 통합
- 높은 기능성
- 정적 타이핑 : 변수 나 함수가 미치 정해진 데이터 형을 처리하도록 설명을 요구 -> 안전하게 추상화가 가능한 타입 시스템 
- Java와의 호환성 " Scala의 소스 파일을 컴파일하면 JVM(Java 가상 머신)의 바이트 코드. Java 가상 머신에서 작동하기 때문에 
Scala에서 Java의 메소드 나 필드를 호출 할 수 있으며, 반대도 가능 
- 병행 처리 : 멀티 코어를 인식하고 효율적으로 병렬 처리를 위한 'Actor'라이브러리 제공 
- 큰 데이터 처리 작업에 적합

단점
- 강력하며 세심한 디자인인 만큼 사용하기 어려움, 높은 학습비용
- 높은 러닝커브
- 컴파일 속도가 느리며 jar 파일 용량이 큼
- 이전 버전 컴파일이 어려움(이전 버전과 호환되지 않는 경우도 다수 존재)

## 4. SAS
장점
- 대용량 데이터를 읽어들이거나 사용하는 데에 용이
- 다른 언어에 비해 배우기 쉽다.
- 파일 관리에 상당한 융통성 : 오라클 DB로부터 데이터 가져오기, 엑셀파일로 저장하기, hwp용으로 편집하기, HTML로 인터넷에 게시하기 등
- 다른 프로그램에 비해 재사용성이 뛰어나다.
- OS 환경에 독립적이다.
- SAS는 주요 소프트웨어의 결합으로 구성되어 있어서 필요한 모듈만 사용할 수 있다.
- 다양한 프로그램을 제공, 분석 결과도 다른 프로그램에 비해 방대

단점
- 비용 : 비싼 가격
- 프로그램이 크다.
- 범용성이 낮아 모두에게 보급하기 어렵다.
- 방대한 분석결과로 인해 분석의 오류를 범하기 쉽다.

## 5. SPSS
장점
- 직관적이고 사용이 쉽다.
- 현존하는 상용 S/W 중 가장 많은 마이닝 알고리즘을 내포
- R과 연계가 가능

단점
- 환경(하드웨어 등)에 따라 속도가 심하게 느려지는 경우가 발생
- 다른 프로그램에 비해 느린 속도 
- 느린 속도로 인해 대용량 자료를 처리하기가 부담
- 시각화 자료들의 품질이 낮음
- 상대적으로 비싼 가격
